Section 01

データ収集設計における技術的・法的制約

主要な口コミ媒体であるGoogle Maps、ホットペッパービューティー、カンナムオンニ、トリビューからのデータ収集においては、プラットフォーム側が定める厳格な利用規約と技術的な取得制限が存在し、無秩序なスクレイピングはコンプライアンス違反のリスクを孕んでいる。

媒体名 API / スクレイピング制約 代替収集手段とサンプリング設計 サンプルサイズ
Google Maps Places APIは最大5件制限 1。バルク保存は規約違反 8。 公開情報の目視確認による手動サンプリング。商用二次データプロバイダ（Scrap.io等）の活用。 30〜50件 / 院
ホットペッパー 競合レビューのAPI取得不可。二次利用・転載制限 13。 公開範囲内での定性的リサーチ。「参考になった」票が多いレビューを目視確認し、要約記録。 上位20〜30件
カンナムオンニ API非公開。クローズドエコシステム。 アプリ内検索による手動サンプリング。特定の施術（糸リフト等）に絞り込み不満傾向を集約。 10〜20件 / 施術

本分析では「星1〜2（顕在化した怒り）」と「星3（期待外れ・潜在的不満）」を明確に区分する。星3には「施術には満足だが待ち時間が長い」といった、クリニックの構造的な課題と改善に直結する建設的な示唆が豊富に含まれている。

Section 02

美容医療特化型NLP前処理パイプライン

ノイズを排除して正確な構造を抽出するため、美容医療ドメインに適合するNLPパイプラインを構築する。一般的な辞書では「未知語」として誤分割される専門用語を正しく認識させる必要がある。

MECHAB + COMEJISYO

専門辞書のコンパイル

医療用語特化辞書「ComeJisyo」をベースに、「エラボトックス」「ダウンタイム」等の語彙をコスト値を低く設定して登録し、誤分割を防止する。

NORMALIZATION

表記ゆれの統一

「ボト」「botox」「ボトックス注射」を「ボトックス」に正規化。ステミング処理により「痛かった」「痛くない」を原形「痛い」に変換する。

MASKING

個人情報の匿名化

正規表現を用いて他院名、医師名、金額を <CLINIC_NAME>, <DOCTOR_NAME>, <PRICE> 等のタグに置換し、感情と構造のみを抽出する。

Section 03

ネガティブ要因の多角的分析

前処理済みのコーパスに対し、TF-IDF、共起ネットワーク、LDA等のアルゴリズムを適用し、不満の根底にある構造を特定する。

TF-IDF法による評価帯域別の特徴

星1・星2：信頼の崩壊

特徴語：「態度」「最悪」「待たされる」「返金」
ホスピタリティの欠如や約束の不履行、契約強要への強い憤りが核となる。

星3：期待値とのギャップ

特徴語：「変わらない」「少し」「高い」「惜しい」
臨床効果が期待値を下回った、あるいは費用対効果への失望が表明される。

感情分析による心理状態のラベリング

Anger — 怒り

ルール不履行への激昂

1時間以上の待機や高圧的な接客により発現。自己拡散力が強く、Google Mapsの低評価レビューに集中する傾向がある。

Anxiety — 不安

説明不足と孤独感

ダウンタイム中のアフターケア欠如に関連して発現。「このまま腫れが引かないのではないか」という術後の孤独感が表れる。

Disappointment — 失望

静かな諦め

事前の期待値と実体験のギャップから生じる。星3レビューに最も多く見られ、費用対効果の悪さを指摘する際に顕著。

施術別の不満構造解剖

ボトックス注射：技術とデザインの失敗

解剖学的知識不足に基づく「機能障害（噛めない）」や「笑顔の引きつり」に不満が集中。注入後2週間でピークに達するため、数ヶ月続く苦痛が星1レビューに直結する。

シミ取り：診断ミスと期待値のズレ

肝斑への不適切な照射による悪化が典型的な診断ミス。ダウンタイム期間やテープ保護の必要性に関する事前認識の甘さがクレームを誘発する。

Section 05

競合の弱点を自院の強みに反転させる施策

01. 待ち時間の撲滅

ダブルブッキングの廃止とメニュー毎の正確な所要時間算出。LINEによる「現在の待ち時間目安」のリアルタイム通知。15分以上の遅延時のリカバリープラン（ドリンク提供等）を徹底する。

02. 医療接遇の高度化

不安を抱える患者の心理に寄り添った言葉遣いやトーンの研修。受付・電話応対のリアルなロールプレイの実施。カスハラ対応基準の策定によるスタッフの心理的負担の軽減。

03. 透明な提案

「ノー・アップセル宣言」の導入。見積もり提示後はスタッフが一度退室し、患者が冷静に検討できる「プレッシャーのない時間」を保証する。段階的注入プランによる失敗リスク最小化。

Section 06

継続運用の仕組みと自動化ダッシュボード

口コミデータは先行指標として定点観測する仕組みが必要である。法的制約を順守したデータパイプラインを構築する。

自動化パイプラインの構成

  • データ取得: 適法なAPIまたはデータベンダーからの月次データ受領。
  • 自動処理: GCP/AWS環境でMeCab形態素解析と匿名化を自動実行。
  • LLM判定: LLM APIを用いて感情極性とカテゴリ、緊急度を自動タグ付け。
  • クリティカルアラート: 「消費者センター」「医療ミス」等のキーワード検出時に即座にSlack通知。
